# Screenshots Directory

This directory contains screenshots of the HYRAX Task Management application interface.

## Screenshot Guidelines

When capturing screenshots for documentation:

### Recommended Screenshots:
- `login.png` - Login interface
- `dashboard-admin.png` - Super Admin dashboard
- `dashboard-manager.png` - Manager dashboard  
- `dashboard-member.png` - Team member dashboard
- `user-management.png` - User management interface
- `add-user.png` - Add user modal
- `campaigns-list.png` - Campaigns listing page
- `campaign-detail.png` - Campaign detail view
- `weekly-view.png` - Weekly task view
- `mobile-view.png` - Mobile responsive interface

### Screenshot Specifications:
- **Resolution**: 1920x1080 (desktop), 375x812 (mobile)
- **Format**: PNG with transparency where applicable
- **Quality**: High resolution, optimized file size
- **Browser**: Chrome/Edge with developer tools (if showing responsive)

### Capture Guidelines:
1. Use realistic data (no lorem ipsum)
2. Show different user roles and permissions
3. Include various task states and statuses
4. Demonstrate key features and workflows
5. Ensure UI is clean and professional

## Tools for Screenshots:
- **macOS**: Command + Shift + 4 (selection), Command + Shift + 3 (full screen)
- **Windows**: Snipping Tool, Print Screen + Paint, or Windows + Shift + S
- **Browser Extensions**: Awesome Screenshot, FireShot
- **Professional Tools**: Lightshot, Greenshot, Skitch

## File Naming Convention:
- Use lowercase with hyphens: `user-management.png`
- Include context: `dashboard-admin.png` vs `dashboard-member.png`
- Version if needed: `login-v2.png`

## Update Process:
1. Capture new screenshots when UI changes significantly
2. Update README.md references to new screenshot files
3. Optimize file sizes before committing
4. Test that all screenshot links work in documentation
